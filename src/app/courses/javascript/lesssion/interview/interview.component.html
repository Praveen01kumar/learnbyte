1. What is javascript?

JavaScript is a high-level, interpreted programming language primarily used for adding interactivity and dynamic behavior to web pages. Initially developed by Brendan Eich at Netscape in 1995, JavaScript has evolved to become one of the most widely used languages on the web.

Here are some key points about JavaScript:

  1. Client-Side Scripting: JavaScript is mainly used as a client-side scripting language, meaning it runs on the user's web browser rather than on the web server. This allows developers to create dynamic web content that responds to user actions without needing to communicate with the server.
  2. Versatile Language: JavaScript is versatile and can be used for a variety of purposes, including building web applications, creating interactive web pages, developing mobile apps (using frameworks like React Native or Ionic), building server-side applications (using Node.js), and more.
  3. Interactivity and Event Handling: JavaScript allows developers to create interactive elements on web pages, such as forms, buttons, animations, and games. It provides APIs for handling user actions like clicks, mouse movements, keyboard input, and touch events.
  4. Dynamic HTML Manipulation: JavaScript can manipulate the HTML and CSS of a web page in real-time. This means you can dynamically change the content, style, and structure of a webpage based on user interactions or other conditions.
  5. Asynchronous Programming: JavaScript supports asynchronous programming, allowing tasks to be executed concurrently without blocking the execution of other code. This is crucial for handling operations like fetching data from servers, processing user input, or performing animations without freezing the user interface.
  6. Cross-Platform Compatibility: JavaScript is supported by all modern web browsers, including Chrome, Firefox, Safari, Edge, and Opera. Additionally, with the rise of server-side JavaScript frameworks like Node.js, JavaScript can also be used for building server-side applications, making it a full-stack language.

Overall, JavaScript is a powerful and flexible language that plays a vital role in modern web development, enabling developers to create rich, interactive web experiences.

2. What are the various data types exist in JavaScript?

JavaScript supports several data types, which can be broadly categorized into two groups: primitive data types and non-primitive (or reference) data types. Here's an overview of each:

  1. Primitive Data Types:
    • Number:Represents numeric values, including integers and floating-point numbers.
    • String:Represents sequences of characters enclosed within single or double quotes.
    • Boolean:Represents logical values, true or false, used for decision-making.
    • Null:Represents the intentional absence of any value.
    • Undefined:Represents a variable that has been declared but not assigned a value.
    • Symbol:Represents unique identifiers, often used as object keys.
  2. Non-Primitive (Reference) Data Types:
    • Object: Represents a collection of key-value pairs where values can be primitive or reference types. Objects are used for storing complex data structures and are the fundamental building blocks of JavaScript. Example: { name: 'John', age: 30 }.
    • Array: Represents a list-like collection of elements. Arrays can contain elements of any data type, including other arrays and objects. Example: [1, 2, 3, 4, 5].
    • Function: Represents a reusable block of code that can be invoked with a set of arguments. Functions are first-class citizens in JavaScript, meaning they can be assigned to variables, passed as arguments, and returned from other functions. Example: function add(a, b) { return a + b; }.
    • Date: Represents a specific point in time, typically expressed as a timestamp. Date objects are used for working with dates and times. Example: new Date().
    • RegExp: Represents a regular expression, used for pattern matching within strings. Regular expressions provide a powerful way to search, match, and replace text. Example: /pattern/.

3. what is truthy and falsy values?

Truthy and falsy values in JavaScript refer to values that are inherently considered as either "true" or "false" when evaluated in a boolean context, such as in conditionals or logical operations. Understanding truthy and falsy values is important because it affects how JavaScript behaves when conditions are evaluated.

Here's a brief explanation along with examples:

  1. Truthy Values:
    • Any value that is not explicitly false, null, undefined, 0, NaN, or an empty string ('') is considered truthy.
    • When evaluated in a boolean context, truthy values result in true.
  2. Falsy Values:
    • Values that are inherently false, empty, or represent absence are considered falsy.
    • When evaluated in a boolean context, falsy values result in false.

Understanding truthy and falsy values is crucial when writing conditions in JavaScript. It allows for concise and readable code by leveraging the implicit boolean coercion of values.

5. What is a prototype chain?

In JavaScript, every object has an internal property called [[Prototype]] (commonly referred to as the "prototype"), which refers to another object. This object is often called the object's prototype, and it forms what is known as the "prototype chain."

Here's how the prototype chain works:

  1. Object Prototype
    • Every JavaScript object has a prototype except for the base object itself, which is the root of the prototype chain.
    • The base object's prototype is null.
  2. Prototype Inheritance
    • When you access a property or method on an object, JavaScript first looks for that property or method on the object itself.
    • If the property or method is not found on the object, JavaScript then looks at the object's prototype (the object referenced by its [[Prototype]] property) and continues the search there.
    • This process continues recursively until the property or method is found or until the end of the prototype chain (i.e., when the [[Prototype]] property of an object is null).
  3. Prototype Chain
    • The prototype chain is essentially a series of linked objects, where each object's prototype is the previous object in the chain.
    • By following this chain, JavaScript can implement inheritance, allowing objects to inherit properties and methods from their prototype objects.

6. What is the difference between Call, Apply and Bind?

call, apply, and bind are methods in JavaScript that are used to manipulate the context (this keyword) of functions. Here's the difference between them:

  1. call
    • The call method is used to invoke a function with a specific this value and arguments provided individually.
    • It takes the this value as the first argument, followed by any additional arguments as needed.
  2. apply
    • Similar to call, the apply method is used to invoke a function with a specific this value and arguments provided as an array or an array-like object.
    • It takes the this value as the first argument and an array (or array-like object) containing the arguments as the second argument.
  3. bind
    • The bind method is used to create a new function with a specified this value and optionally pre-filled arguments.
    • It does not immediately invoke the function but returns a new function with the specified context and arguments.

In summary:

  • Use call when you want to pass arguments individually.
  • Use apply when you want to pass arguments as an array or array-like object.
  • Use bind when you want to create a new function with a specified context and optionally pre-filled arguments, without immediately invoking the function.

7. what is "this" in Javascript?

In JavaScript, this is a special keyword that refers to the object on which a method is being invoked, or in other words, the context in which a function is called. The value of this depends on how a function is called, and it is determined dynamically at runtime. Understanding this is crucial for working with object-oriented JavaScript and for maintaining the correct context within functions.

Here are some key points about this:

  1. Global Context
    • In the global scope (outside of any function), this refers to the global object.
    • In a web browser environment, the global object is window.
  2. Function Context
    • Inside a function, the value of this depends on how the function is invoked.
    • If a function is called as a method of an object, this refers to the object itself.
  3. Constructor Context
    • When a function is used as a constructor with the new keyword, this refers to the newly created instance of the object.
  4. Explicit Binding
    • You can explicitly set the value of this using call, apply, or bind methods.
    • This allows you to control the context in which a function is called.
  5. Arrow Functions
    • Arrow functions do not have their own this context. Instead, they inherit this from the surrounding lexical scope (the context in which they are defined).

Understanding how this behaves in different contexts is essential for writing effective and maintainable JavaScript code.

9. What is the purpose of the array slice method?

The slice() method in JavaScript is used to extract a section of an array and return a new array containing the extracted elements. It does not modify the original array; instead, it returns a shallow copy of a portion of the array.

Here's the purpose of the slice() method along with examples:

  1. Extracting a Range of Elements

    You can use slice() to extract a range of elements from an array. Provide the starting index (inclusive) and the ending index (exclusive).

  2. Copying an Entire Array

    You can use slice() without any arguments to create a shallow copy of the entire array.

  3. Extracting the Last n Elements

    You can use negative indices to extract elements starting from the end of the array.

  4. Extracting Elements from a Specific Index to the End

    You can omit the second argument to extract elements from a specific index to the end of the array.

  5. Using Slice with Strings

    slice() can also be used with strings to extract substrings based on character indices.

  6. Extracting Elements from an Empty Array

    slice() can be used on an empty array to return a new empty array.

The slice() method is versatile and useful for extracting and copying portions of arrays and strings without modifying the original data.

10. What is the purpose of the array splice method?

The splice() method in JavaScript is used to change the contents of an array by removing or replacing existing elements and/or adding new elements in place. Unlike the slice() method, which returns a new array containing a portion of the original array, splice() modifies the original array directly.

Here's the syntax of the splice() method:

  • startIndex: The index at which to start changing the array.
  • deleteCount: The number of elements to remove from the array (starting from startIndex). If set to 0, no elements will be removed.
  • item1, item2, ...: Optional items to add to the array at startIndex.

Here are some examples of using the splice() method:

  1. Removing Elements

    Remove elements starting from a specified index (startIndex) and delete a certain number of elements (deleteCount).

  2. Replacing Elements

    Replace elements starting from a specified index (startIndex) and delete a certain number of elements (deleteCount), then insert new elements.

  3. Adding Elements

    Add new elements to an array at a specific index (startIndex) without removing any elements (deleteCount set to 0).

  4. Removing and Returning Elements

    You can use splice() to both remove elements and return them as a new array.

The splice() method is versatile and allows for various manipulations of array contents, including removal, replacement, and insertion, all within the same operation.

11. what is the difference between slice and splice?

The main difference between slice() and splice() in JavaScript lies in how they modify the original array:

  1. slice()
    • Returns a new array containing a shallow copy of a portion of the original array.
    • Does not modify the original array; it creates and returns a new array with selected elements.
    • The original array remains unchanged.
  2. splice()
    • Modifies the original array by removing, replacing, or adding elements in place.
    • Returns an array containing the removed elements (if any).
    • Changes the original array directly.

In summary, slice() is used to create a new array from a portion of the original array without modifying the original array, while splice() is used to modify the original array by removing, replacing, or adding elements in place.

12. How do you compare Object and Map?

Object and Map are both commonly used data structures in JavaScript, but they serve different purposes and have different characteristics. Here's a comparison between the two:

  1. Purpose
    • ObjectObjects are used to store key-value pairs where keys are strings (or symbols) and values can be of any data type. They are used for representing structured data and are fundamental in JavaScript.
    • MapMaps are also used to store key-value pairs, but unlike objects, the keys in maps can be of any data type, not just strings or symbols. They offer more flexibility in terms of the types of keys that can be used.
  2. Key Type
    • ObjectKeys must be strings or symbols.
    • MapKeys can be of any data type, including objects, functions, and primitive values.
  3. Order
    • ObjectThe order of properties in an object is not guaranteed. Properties may be iterated in the order they were added, but this behavior is not standardized.
    • MapThe order of key-value pairs is preserved. When iterating over a map, the order of insertion is maintained.
  4. Size
    • ObjectTo get the number of properties in an object, you need to manually iterate over the object or use Object.keys(obj).length, which can be inefficient for large objects.
    • MapProvides a size property to directly get the number of key-value pairs in the map.
  5. Iterating
    • Object Iterating over properties of an object can be done using for...in loop or Object.keys, Object.values, or Object.entries methods.
    • MapIterating over a map can be done using methods like forEach, keys, values, or entries.
  6. Performance
    • ObjectGenerally optimized for property access and manipulation. Property access is usually faster compared to maps.
    • MapOptimized for scenarios where keys are frequently added or removed, especially with large datasets.
  7. Memory Usage
    • ObjectMay consume less memory for smaller datasets due to internal optimizations by JavaScript engines.
    • MapMay consume more memory compared to objects, especially for larger datasets, because of the additional overhead associated with each entry.

In summary, use Object when you have string or symbol keys and don't need to preserve insertion order or support keys of different data types. Use Map when you need more flexibility in the types of keys, need to preserve insertion order, or when working with large datasets where efficient addition and removal of key-value pairs are crucial.

29. What are modules and Why do you need modules?

Modules in programming languages, including JavaScript, are essentially files that contain reusable code or units of functionality. They help organize code into smaller, more manageable pieces, making it easier to maintain, reuse, and collaborate on larger projects. JavaScript modules were introduced as part of ES6 (ECMAScript 2015) to provide a standardized way to encapsulate code and its dependencies.

Here's why modules are important and why you need them:

  1. EncapsulationModules encapsulate related code into separate units, allowing you to hide implementation details and expose only what's necessary. This helps reduce complexity and makes code easier to understand and maintain.
  2. Code OrganizationModules allow you to organize your codebase into logical units based on functionality, making it easier to navigate and manage larger projects. Each module can focus on a specific task or feature, improving code readability and maintainability.
  3. ReusabilityModules promote code reuse by allowing you to import and use functionality from one module into another. This reduces duplication of code and promotes a more modular and DRY (Don't Repeat Yourself) coding approach.
  4. Dependency ManagementModules help manage dependencies between different parts of your application. By explicitly specifying dependencies, modules make it easier to understand and track the dependencies between different components, libraries, or modules.
  5. IsolationModules provide a level of isolation, ensuring that variables and functions defined within a module do not pollute the global scope. This helps prevent naming collisions and conflicts between different parts of your codebase.
  6. ScalabilityAs your project grows, modules make it easier to scale and maintain your codebase. You can add new modules or refactor existing ones without impacting the rest of your application, allowing for more agile development and easier collaboration among team members.

Overall, modules are essential for building maintainable, scalable, and modular JavaScript applications. They promote best practices in code organization, encapsulation, and dependency management, leading to more robust and easier-to-maintain codebases.

53. What are the main rules of promise?

A promise must follow a specific set of rules:

i. A promise is an object that supplies a standard-compliant .then() method

ii. A pending promise may transition into either fulfilled or rejected state

iii. A fulfilled or rejected promise is settled and it must not transition into any other state.

iv. Once a promise is settled, the value must not change.

58. What is a strict mode in javascript?

Strict Mode is a new feature in ECMAScript 5 that allows you to place a program, or a function, in a “strict” operating context. This way it prevents certain actions from being taken and throws more exceptions. The literal expression "use strict"; instructs the browser to use the javascript code in the Strict mode.

59. Why do you need strict mode?

Strict mode is useful to write "secure" JavaScript by notifying "bad syntax" into real errors. For example, it eliminates accidentally creating a global variable by throwing an error and also throws an error for assignment to a non-writable property, a getter-only property, a non-existing property, a non-existing variable, or a non-existing object.
